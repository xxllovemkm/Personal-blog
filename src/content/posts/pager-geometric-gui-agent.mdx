## 一句话概括

PAGER 研究的是 GUI agent 在几何构造中的点级精确控制：模型知道该做什么还不够，必须能在连续画布上准确点到正确位置。

## 论文要解决的问题

现有 GUI agent 很多任务是区域容错的。点击按钮、选择菜单、填写输入框时，只要动作落在同一个组件区域内，通常都算成功。这样的设定适合网页和移动应用，但不适合几何构造。

几何画布里的点、线、圆和多边形依赖精确坐标。一个点偏了几个像素，后续连线、交点、角度和拓扑关系都会连锁错误。论文把这类任务定义为 precision-sensitive GUI tasks。

这带来一个重要观察：模型可能已经理解语义步骤，动作类型准确率也很高，但任务成功率仍然很低。原因在于语义规划和像素执行之间存在鸿沟。

## 核心思路

PAGER 从两个层面处理这个问题：

- PAGE Bench：构建点级几何 GUI 控制基准，包含 4,906 个问题和超过 224K 条过程监督的像素级 GUI 动作。
- PAGER agent：用拓扑感知规划管理构造依赖，再用像素级执行完成具体操作。

论文强调几何原语具有 ontological dependencies。也就是说，后续对象不是独立动作，而是建立在前面对象之上。一个早期点的误差会改变整个构造图的拓扑。

## 方法拆解

PAGER 先通过 pixel-grounded supervised tuning 建立可执行动作语法，让模型学会把语义动作落到 GUI 操作上。随后用 precision-aligned reinforcement learning 缓解 rollout 中的 exposure bias。

这里的强化学习反馈不是泛泛的任务奖励，而是状态条件下的几何反馈。它让模型在执行过程中意识到精度偏差如何影响后续结构。

论文报告了一个很有冲击力的现象：通用多模态模型可以达到超过 88% 的动作类型准确率，但任务成功率低于 6%。PAGER 将最强通用 baseline 的任务成功率提升 4.1 倍，并把 GUI-specialized agents 的 step success rate 从低于 9% 提升到超过 62%。

## 为什么重要

这篇论文说明 GUI agent 的难点不能只用“看懂页面”和“选对动作类型”概括。很多真实环境要求精确执行，尤其是设计工具、CAD、标注平台、科学绘图和几何软件。

如果 agent 不能处理点级精度和依赖传播，它就很难从简单网页自动化走向专业生产工具。PAGER 把 GUI 控制问题推进到更接近真实工程任务的层面。

## 我的解读

PAGER 和 GGBench 构成了很自然的上下游关系。GGBench 测试模型能否理解并生成几何构造，PAGER 则进一步要求模型在 GUI 环境中把构造执行出来。前者强调生成式推理，后者强调语义到执行的落地。

我认为“Semantic-Execution Gap”是 GUI agent 方向非常重要的概念。很多模型 demo 看起来会操作界面，但一旦任务需要高精度、长依赖和可验证结果，问题就会暴露。PAGER 的意义在于把这种差距量化，并提供了一个可训练、可评测的闭环方案。

## 可以继续追的问题

- 点级几何控制能否迁移到 CAD、绘图软件和科学可视化工具？
- 拓扑感知规划是否可以和显式几何求解器结合？
- GUI agent 的 reward 应该如何同时考虑语义正确、坐标准确和后续可恢复性？
- PAGER 的过程监督能否用于训练更通用的 precision-sensitive agent？
